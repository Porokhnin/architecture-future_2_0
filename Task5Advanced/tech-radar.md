# Технологический радар холдинга «Будущее 2.0» (Target Tech Radar)

Технологический радар структурирован по четырем ключевым квадрантам и отражает целевое видение архитектуры, паттернов разработки и инфраструктурного стека компании на ближайшие 3 года.

---

## Квадрант 1: Архитектурные паттерны и подходы (Patterns & Methodologies)

Статус жизненного цикла технологий на радаре холдинга:
* Adopt (Внедрение): Технологии и паттерны, обязательные к использованию. Прошли проверку ИБ и ИТ-комитета холдинга.
* Trial (Тестирование): Рекомендованы для пилотных проектов в одном-двух доменах холдинга для оценки влияния на инфраструктуру и TCO (стоимость облака).
* Assess (Оценка): Технологии, изучаемые архитектурным комитетом под будущие цели (например, выход на зарубежные рынки).
* Hold (Заморозка/Запрет): Устаревшие технологии (legacy), создающие риски комплаенса и безопасности, либо паттерны, замедляющие Time-to-Market холдинга.
---


| Технология / Паттерн | Статус | Описание и обоснование применения в холдинге |
| :--- | :--- | :--- |
| **Data Mesh** | **Adopt** | Децентрализация владения данными по бизнес-доменам (Медицина, Финтех, Фарма). Каждый домен самостоятельно упаковывает данные и предоставляет их в виде продукта через контракты API/Событий. Ликвидирует бутылочное горлышко центрального DWH. |
| **Event-Driven Architecture (EDA)** | **Adopt** | Паттерн асинхронного междоменного взаимодействия через события. Устраняет жесткую связность систем на критическом пути. Позволяет подключать новые домены (Фарма, Электроника) за дни без изменения логики старых. |
| **Self-Service BI** | **Adopt** | Концепция портала самообслуживания. Пользователи на местах конструируют кастомные отчеты любой сложности по правилам своего уровня доступа (RLS), не привлекая ИТ-отдел и дата-инженеров. |
| **Transactional Outbox (via CDC)** | **Adopt** | Архитектурный паттерн гарантированной публикации доменных событий. Запись бизнес-сущности и события изменения состояния происходит в рамках одной транзакции в OLTP БД, а инструмент CDC асинхронно доставляет его в брокер. |
| **Microservices (Domain-Driven Design)** | **Adopt** | Распил монолитной бизнес-логики (хранимых процедур SQL Server и интерфейса PowerBuilder) на легковесные изолированные сервисы с собственной базой данных (Database-per-Service). |
| **CQRS** | **Trial** | Разделение операций чтения и записи (Command Query Responsibility Segregation) для высоконагруженных финтех-сервисов и медицинского расписания. |
| **Data Contracts (Avro/Protobuf)** | **Trial** | Жесткая фиксация схем обмена данными между доменами холдинга с обязательной проверкой обратной совместимости. |
| **Monolithic DWH Architecture** | **Hold** | **ЗАПРЕТ.** Подход с единым гигантским транзакционно-аналитическим хранилищем признан неэффективным, блокирующим Time-to-Market и нарушающим ИБ-комплаенс. |
| **Synchronous Request-Response (RPC) on Critical Path** | **Hold** | **ЗАПРЕТ.** Синхронные блокирующие вызовы между доменами холдинга на сквозных процессах запрещены для предотвращения каскадных падений. |

## Квадрант 2: Платформы данных и Аналитика (Data Platforms & Analytics)


| Технология / Паттерн | Статус | Описание и обоснование применения в холдинге |
| :--- | :--- | :--- |
| **ClickHouse** | **Adopt** | Основная колоночная аналитическая СУБД (OLAP) холдинга. Идеально подходит для мгновенной агрегации сотен терабайт деобезличенных данных и обслуживания портала самообслуживания в режиме Near-Real-Time. |
| **PostgreSQL / YDB** | **Adopt** | Целевой стандарт для всех доменных операционных баз данных (OLTP). PostgreSQL используется для биллинга, медкарт и авторизации взамен легаси SQL Server 2008. |
| **dbt (data build tool) / Cube.js** | **Adopt** | Инструменты создания семантического слоя данных. Скрывают от бизнес-пользователей портала BI физическую структуру таблиц БД, заменяя их на понятные бизнес-термины (выручка, пациентский поток). |
| **Apache Superset** | **Adopt** | Современная open-source BI-платформа, развернутая в качестве портала самообслуживания. Поддерживает Drag-and-Drop конструктор отчетов, Row-Level Security (RLS) на уровне строк и бесшовно работает с ClickHouse. |
| **Apache Flink / Spark Streaming** | **Trial** | Движки потоковой аналитики и трансформации данных «на лету» прямо из очередей Kafka для расчета финансовых и операционных KPI холдинга с задержкой < 5 секунд. |
| **Microsoft SQL Server 2008** | **Hold** | **ВЫВОД ИЗ ЭКСПЛУАТАЦИИ.** Легаси-СУБД, находящаяся вне поддержки. Полный запрет на использование в новых сервисах. Текущие данные выгружаются строго асинхронно через CDC и утилизируются. |
| **Power BI (Кастомный поверх DWH)** | **Hold** | **ЗАМОРОЗКА.** Текущие кастомные отчеты консервируются на этапе миграции. Создание новых отчетов в старой системе запрещено; аналитика переносится на Apache Superset. |

## Квадрант 3: Интеграция и Инфраструктура (Integration & Infrastructure)


| Технология / Паттерн | Статус | Описание и обоснование применения в холдинге |
| :--- | :--- | :--- |
| **Apache Kafka** | **Adopt** | Центральный нервный узел холдинга. Распределенный, горизонтально масштабируемый брокер сообщений, обеспечивающий асинхронный амортизационный буфер (Backpressure) между доменами. |
| **Kubernetes (K8s)** | **Adopt** | Целевая облачная платформа для оркестрации контейнеров приложений (микросервисов) всех доменов холдинга, гарантирующая автопилотирование ресурсов и быстрое развертывание в новых регионах. |
| **Debezium (Kafka Connect)** | **Adopt** | Основной инструмент реализации CDC (Change Data Capture). Асинхронно считывает журналы транзакций доменных БД (включая легаси SQL Server 2008), не нагружая CPU баз данных и не создавая блокировок. |
| **Confluent Schema Registry** | **Adopt** | Сервис централизованного хранения контрактов (Avro/Protobuf-схем). Блокирует публикацию некорректных сообщений в Kafka, защищая домены-подписчики от падения. |
| **Terraform / Helm** | **Adopt** | Реализация подхода Infrastructure as Code (IaC). Позволяет развернуть ИТ-контур холдинга в новом географическом регионе методом копирования конфигурационных скриптов. |
| **Keycloak (OAuth2 / OIDC / SSO)** | **Adopt** | Единая платформа аутентификации холдинга. Отвечает за верификацию пользователей (Врачи, Администраторы, Клиенты банка) и сквозную передачу ролей безопасности. |
| **Apache Camel (ESB)** | **Hold** | **ЗАМОРОЗКА.** Старая синхронная шина данных переводится в статус «Мост совместимости». Новые интеграции на Camel запрещены. Полный вывод из эксплуатации к 18-му месяцу трансформации. |
| **PowerBuilder** | **Hold** | **ЗАПРЕТ.** Десктопный интерфейс операторов клиник полностью выводится из эксплуатации. Логика переносится в Web-микрофронтенды. |

## Квадрант 4: Языки и Инструменты разработки (Languages & Frameworks)


| Технология / Паттерн | Статус | Описание и обоснование применения в холдинге |
| :--- | :--- | :--- |
| **Go (Golang)** | **Adopt** | Основной целевой язык для создания высоконагруженных финтех-сервисов (процессинг, кредитование) и критичных медицинских API (сервис расписания) из-за низкого потребления памяти и высокой скорости. |
| **Java (Spring Boot / .NET Core)** | **Adopt** | Целевой стек для разработки бизнес-микросервисов со сложной доменной логикой (ведение электронных медицинских карт, биллинг клиник). |
| **Python (FastAPI / PyTorch / TensorFlow)**| **Adopt** | Единый стандарт для домена `MedAI`. Используется для создания легковесных оркестраторов моделей и воркеров инференса медицинских DICOM-снимков. |
| **React / Vue.js** | **Adopt** | Стандарт для фронтенд-разработки. Заменяет PowerBuilder на современные веб-интерфейсы и АРМ специалистов, работающие через API Gateway. |
| **T-SQL Stored Procedures** | **Hold** | **ЗАПРЕТ.** Категорический запрет на написание бизнес-логики внутри баз данных. База данных должна быть исключительно слоем хранения состояния (Stateful), вся логика выносится в Stateless микросервисы в K8s. |
