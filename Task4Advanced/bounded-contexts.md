## Ограниченные контексты

### Ограниченный контекст: `MedicalCore` (Операционная медицина)
Ядро автоматизации клиник холдинга. Ответственность: ведение пациентского потока, истории болезни, расписания специалистов и диагностических назначений.

*   **Ключевые Агрегаты (Границы и Инварианты):**
    *   **`PatientProfile` (Профиль пациента)**
        *   *Корень агрегата (Root Entity):* `Patient`
        *   *Идентификатор:* `PatientHashID` (UUID-хэш).
        *   *Инвариант:* Профиль не может быть сохранен в системе без генерации `PatientHashID` (хэш `SHA-256` на основе паспорта и ФИО с секретной солью). Персональные данные пациентов физически заперты внутри этого агрегата.
    *   **`Appointment` (Медицинский прием)**
        *   *Root Entity:* `MedicalAppointment`
        *   *Идентификатор:* `AppointmentID` (UUID)
        *   *Инвариант:* Статус приема не может быть переведен в состояние «Завершен» без прикрепленного электронного протокола осмотра и уникального ID лечащего врача.
    *   **`DiagnosticStudy` (Медицинское исследование)**
        *   *Root Entity:* `Study`
        *   *Идентификатор:* `StudyID` (UUID)
        *   *Инвариант:* Ссылка на файл исследования в S3-хранилище становится доступной для внешних систем только после валидации DICOM-заголовков на целостность данных.
*   **Важные междоменные события:**
    *   `PatientRegistered` — зарегистрирован новый пациент (слушает: `GlobalAnalytics`).
    *   `StudyUploaded` — загружены результаты исследования (слушает: `MedAI` для запуска нейросетей).
    *   `MedicalInvoiceRequested` — сформирована необходимость оплаты медицинских услуг (слушает: `FintechCore`).

### Ограниченный контекст: `FintechCore` (Банкинг и Биллинг)
Инкапсулирует операции приобретенного банка холдинга. Ответственность: расчет стоимости приемов, проведение транзакций, оценка рисков (скоринг) и целевое кредитование.

*   **Ключевые Агрегаты (Границы и Инварианты):**
    *   **`Invoice` (Медицинский инвойс / Счет)**
        *   *Root Entity:* `Invoice`
        *   *Идентификатор:* `InvoiceID` (UUID)
        *   *Инвариант:* Итоговая сумма к оплате должна строго равняться сумме всех позиций оказанных услуг за вычетом примененных скидок холдинга. Оплаченный инвойс полностью блокируется для любых изменений.
    *   **`CreditContract` (Кредитный договор / Рассрочка)**
        *   *Root Entity:* `CreditAgreement`
        *   *Идентификатор:* `AgreementID` (Строковый номер банковского договора)
        *   *Инвариант:* Договор рассрочки на лечение не может быть переведен в статус «Активирован» без прохождения скоринга платежеспособности и без валидной привязки к активному `InvoiceID` из контекста медицины.
*   **Важные междоменные события:**
    *   `InvoiceIssued` — пациенту официально выставлен счет на оплату услуг (слушает: `GlobalAnalytics`).
    *   `CreditAgreementCreated` — создан и подписан кредитный договор под медицинские услуги холдинга.
    *   `PaymentProcessed` — успешно проведена транзакция оплаты (слушает: `MedicalCore` для автоматического закрытия приема врача и смены статуса инвойса).

### Ограниченный контекст: `MedAI` (ИИ-Диагностика)
Поддерживающий контекст, отвечающий за оркестрацию и инференс нейросетевых моделей для автоматического распознавания патологий на медицинских снимках.

*   **Ключевые Агрегаты (Границы и Инварианты):**
    *   **`AIInference` (Сессия инференса)**
        *   *Root Entity:* `InferenceJob`
        *   *Идентификатор:* `InferenceID` (UUID)
        *   *Инвариант:* Анализ снимка запускается строго на той версии ИИ-модели, которая зафиксирована в реестре медицинских алгоритмов на дату исследования. Метрика уверенности модели (`ConfidenceScore`) должна находиться в диапазоне `[0.00, 1.00]`.
*   **Важные междоменные события:**
    *   `InferenceCompleted` — завершено исследование ИИ. Содержит булево поле `anomalyDetected` и `confidenceScore` (слушают: `MedicalCore` для вывода подсказки в АРМ врача; `GlobalAnalytics` для контроля качества моделей холдинга).

### Ограниченный контекст: `IdentityAccess` (Сквозной контекст безопасности)
Инфраструктурный контекст (Generic Subdomain), инкапсулирующий Keycloak и кастомный микросервис сессий `auth`.
*   **Назначение:** Проверяет права пользователей, генерирует JWT-токены, управляет сквозным Row-Level Security (RLS) для портала самообслуживания.
