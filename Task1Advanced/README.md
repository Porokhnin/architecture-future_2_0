# Модульная инфраструктура виртуальных машин

Универсальный переиспользуемый модуль для развертывания ВМ в Yandex Cloud под разные окружения (`dev`, `stage`, `prod`).

## Входные параметры модуля (`modules/vm`)

Модуль принимает следующие 5 параметров:
1. `cores` (number) — Количество ядер.
2. `memory` (number) — Объём RAM (в ГБ).
3. `boot_disk_size` (number) — Подключаемый диск (размер в ГБ).
4. `subnet_id` (string) — Subnet ID.
5. `ssh_key` (string) — SSH-ключ.

## Выходные данные модуля (`outputs.tf`)
* `instance_id` — Уникальный ID виртуальной машины.
* `external_ip` — Публичный IP-адрес ВМ.
* `disk_id` — ID подключаемого загрузочного диска.

## Как запустить для конкретного окружения (на примере dev)

1. Перейдите в директорию нужного окружения:
   ```bash
   cd envs/dev/
   ```
2. Инициализируйте Terraform и загрузите модуль:
   ```bash
   terraform init
   ```
3. Проверьте план изменений с использованием конфигурационного файла среды:
   ```bash
   terraform plan -var-file="terraform.tfvars"
   ```
4. Примените конфигурацию для развертывания:
   ```bash
   terraform apply -var-file="terraform.tfvars"
   ```
